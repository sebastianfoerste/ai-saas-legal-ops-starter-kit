# Policy: AI Vendor Governance & Tool Usage

## 1. Objective
This policy defines the permitted, restricted, and prohibited use cases for third-party artificial intelligence (AI) systems, large language models (LLMs), and automated APIs.

## 2. Governance Information
*   **Policy Owner**: General Counsel (`sebastianfoerste`)
*   **Security Lead**: CISO
*   **Review Cadence**: Semi-Annually (Every 6 months)
*   **Last Updated**: 2026-06-01

## 3. Usage Classifications

### A. Permitted Use
The following use cases are approved without case-by-case review, provided they use **approved tools** (e.g. enterprise-grade models with zero data retention):
*   Drafting boilerplate public documentation, blog drafts, and marketing emails.
*   Anonymized code suggestions and software debugging.
*   Summarizing generic, publicly-available research papers.

### B. Restricted Use
The following use cases require completing an **AI Vendor Review** intake and receiving written approval from the Privacy or Security lead:
*   Inputting customer support chat transcripts or feedback logs.
*   Processing internal company slide-decks containing strategic roadmaps.
*   Using automated tools to draft customer contracts or response briefs.

### C. Prohibited Use
The following uses are strictly banned. No exceptions:
*   Entering customer Personal Identifiable Information (PII) or HIPAA-covered health records into public, un-gated AI tools.
*   Entering proprietary source code or trade secrets into tools that train their base models on user inputs.
*   Using AI models to make automated, un-reviewed decisions affecting employee status or creditworthiness.

## 4. Approved Tool Registry
All tools must be listed in our Approved Tool Registry. Approved tools must maintain SOC 2 Type II compliance and contractually opt-out of model training.
